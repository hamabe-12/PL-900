# <a name="lab-1-data-modeling"></a>ラボ 1:データ モデリング

## <a name="scenario"></a>シナリオ

ベローズ カレッジは、カレッジ内に複数の建物を持つ教育機関です。 カレッジ訪問は現在、紙の記録簿に記録されています。 その情報は一貫して把握されておらず、カレッジ全体の訪問に関するデータを収集して分析する手段もありません。

カレッジの管理者は、建物へのアクセスがセキュリティ担当者によって管理され、すべての訪問者がホストによって事前に登録され、記録されることが要求される訪問者登録システムを近代化したいと考えています。

このコース全体を通して、アプリケーションを構築するとともに自動化を行って、ベローズ カレッジの管理担当者とセキュリティ担当者がキャンパス内の建物へのアクセスを管理および制御できるようにします。

このラボでは、環境をセットアップし、Microsoft Dataverse データベースを作成し、変更を追跡するソリューションを作成します。 また、次の要件をサポートするデータ モデルも作成します。

- R1 - スケジュールされたキャンパス訪問の情報を追跡します

- R2 – 訪問者を識別し、追跡するための基本情報を記録します

- R3 – 訪問をスケジュール、記録、および管理します


## <a name="high-level-lab-steps"></a>ラボ手順の概要

学習環境を準備するには、次の手順を実行します。

- メタデータの説明 (テーブルおよびリレーションシップ) については、[データ モデルのドキュメント](https://github.com/MicrosoftLearning/PL-900-Microsoft-Power-Platform-Fundamentals/blob/master/Allfiles/Campus%20Management.png)を参照してください。 Ctrl キーを押しながらクリックするか、リンクを右クリックして、データ モデル ドキュメントを新しいウィンドウで開くことができます。
- Excel スプレッドシートを使用して訪問テーブルを作成する

## <a name="exercise-1-create-new-table"></a>演習 1:新しいテーブルを作成する

**目的:** このテーブルでは、新しいカスタム テーブル "訪問" を作成します。

### <a name="task-1-visit-file-download"></a>タスク \#1:Visits.xlsx ファイルをダウンロードする

**訪問** テーブルには、訪問者、スケジュールされた時間、各訪問の実際の時間などのキャンパス訪問に関する情報が含まれます。

訪問チェックイン プロセス中に尋ねられたときに、訪問者が簡単に入力して解釈できる一意の番号を毎回ごとの訪問に割り当てたいと思います。

このタスクでは、タスク \#2でテーブル作成する際に使用するデータをダウンロードします。

1. **Visits.xlsx** ファイルがデスクトップに格納されている必要があります。遷移ページ [Visits.xlsx](file/Visits.xlsx) の右にある [ダウンロード] ボタン をクリックするとダウンロードできます。

### <a name="task-1-create-visit-table-and-columns"></a>タスク \#2:訪問テーブルを作成する

1. [https://make.powerapps.com](https://make.powerapps.com/) にサインインします (まだサインインしていない場合)

2. **[[自分のイニシャル] 練習]** 環境(例：TK練習)がまだ選択されていない場合は、右上で選択します。

3. 左側のナビゲーションを使用して **[テーブル]** を選択します。

4. **[新しいテーブルを作成する]** をクリック、**[Excelファイル または .CSVファイルをインポート]** を選択します。

5. **[デバイスから選択]** ボタンをクリックします。

6. タスク \#1でダウンロードした **[Visits.xlsx]** ファイルを見つけて選択します。  (このファイルのアップロードには、1 から 2 分かかる場合があります。）

7. Visits.xlsxのExcleファイルから **[イベントスケジュール]** テーブルにインポート済みの画面が表示され、**[イベントスケジュール]** の右側にある **[...]** をクリック、 **[プロパティ]** を選択すると、テーブルの編集画面が表示されて、下記に示す通りに編集を行い、**[保存]** をクリックします。

    - **[表示名]** に「**訪問**」を入力します。
    
    - **[プライマリー列]** を「**name**」列に変更します。

    - **[高度なオプション]** を展開、**[スキーマ名]** に「**Visitor**」を入力します。

8. 再度 **[イベントスケジュール]** の右側にある **[...]** をクリック、 **[データを表示する]** を選択すると、各列のデータが表示され、下記に示す通りに編集を行います。

9. **[イベント名]** 列を編集します。

    - **[イベント名]** をクリックします。
    
    - **[列の編集]** をクリックします。
    
    - **[表示名]** に「**Name**」を入力します。
    
    - **[必須]** を **[ON]** に変更します。
    
    - 高度なオプションを展開、**[スキーマ名]** に「**Name**」を入力します。（表示名の変更により、既に入力される場合に変更なしです）
    
    - **[更新]** ボタンをクリックします。
    
10. **[コード]** 列を編集します。

    - **[コード]** をクリックします。
    
    - **[列の編集]** をクリックします。
    
    - **[必須]** を **[ON]** に変更します。
    
    - **[更新]** ボタンをクリックします。
    
11. **[予定開始]** 列を編集します。

    - **[予定開始]** をクリックします。
    
    - **[列の編集]** をクリックします。
    
    - **[表示名]** に「**開始予定**」を入力します。
    
    - **[必須]** を **[ON]** に変更します。
    
    - 高度なオプションを展開、**[スキーマ名]** に「**scheduledstart**」を入力します。
    
    - **[更新]** ボタンをクリックします。
    
12. **[予定終了]** 列を編集します。

    - **[予定終了]** をクリックします。
    
    - **[列の編集]** をクリックします。
    
    - **[表示名]** に「**終了予定**」を入力します。
    
    - **[必須]** を **[ON]** に変更します。
    
    - 高度なオプションを展開、**[スキーマ名]** に「**scheduledend**」を入力します。
    
    - **[更新]** ボタンをクリックします。
    
13.右上の **[保存して終了]** ボタンをクリックします。ポップアップが出て、再度 **[保存して終了]** ボタンをクリックします。
    少し時間を経ったら、テーブル一覧画面に戻り、**[訪問]** テーブルをクリックして、作成された [訪問] テーブル画面が表示され、そして Visits.xlsx データが [訪問 列とデータ] セクションに表示されていることを確認できます。
    
### <a name="task-1-create-visit-table-and-columns"></a>タスク \#3:列を編集・追加を作成する

> 訪問の時刻は常に建物の場所に対してローカルであり、異なるタイム ゾーンから表示しても変更してはならないため、 **[タイム ゾーン非依存]** 動作を使用して、日付と時刻の情報を記録します。

1. **[開始予定]** 列を編集します。

    - **[開始予定]** をクリックします。
    
    - **[列の編集]** をクリックします。
    
    - **[検索可能]** のチェックボックスをクリック、「**✔**」を入力します。
    
    - 高度なオプションを展開、**[タイムゾーンの調整]** を「**タイムゾーン非依存**」に変更します。
    
    - **[保存]** ボタンをクリックします。
    
2. **[終了予定]**、**[実際の開始]**、**[実際の終了]** 列も1.の手順通りに **[検索可能]** の「**✔**」入れと 「**タイムゾーン非依存**」変更及び **[保存]** を行います。

3. **[コード]** 列を編集します。

    - **[コード]** をクリックします。
    
    - **[列の編集]** をクリックします。
    
    - **[データの種類]** で「**オートナンバー**」を選択します。
    
    - **[オートナンバーの種類]** に 「**日付が先頭に付加される数**」を選択します。
    
    - **[保存]** ボタンをクリックします。
    
4. **[訪問者]** の検索列を作成します。

    - 左上にある **[＋新規]** をクリックします。
    
    - **[列]** をクリックします。
    
    - **[表示名]** に「**訪問者**」を入力します。
    
    - **[データの種類]** で **[検索]** - **[検索]** を選択します。

    - **[関連テーブル]** で **[取引先担当者]** を選択します。

    - **[高度なオプション]** を展開します。

    - **[スキーマ名]** に「**Visitor**」を入力します。

    - **[リレーションシップ名]** に「**visitor_id**」に変更します。
    
    - **[保存]** ボタンをクリックします。

これで、正常に新しいテーブルが作成されました。
